## 2024-05-15 - Enhancing Form Accessibility with useId()
**Learning:** Found that custom form components like `Input.tsx` were missing explicit connection between labels, inputs, helper text, and error messages. Screen readers require `htmlFor` on labels and `aria-describedby` on inputs to understand these relationships properly, especially for dynamically rendered messages or multiple instances of the same component.
**Action:** Used React 18's `useId()` hook to auto-generate unique, accessible IDs for the input, helper text, and error message elements. Linked them together using `htmlFor` and `aria-describedby` attributes, and added `aria-invalid` to programmatically communicate validation states. Applied this pattern to the `Input` component as a reusable baseline for other form elements in the design system.
